Kitview Search v5.2 Simplifié

Guide d'utilisation de l'interface simplifiée (simple30.html)

📋 1. Présentation

Kitview Search — Version Simplifiée est une interface épurée de recherche multilingue permettant d'interroger une base de données de patients orthodontiques. Tous les contrôles essentiels (base, langue, détection) sont directement visibles dans le bandeau, sans barre d'outils à activer. L'interface s'adapte à tous les écrans (PC, tablette, mobile).

💡 Interface simplifiée

Par rapport à la version complète (chat30.html), cette version supprime : le mode démo, la page paramètres, la page analyse et la checkbox de barre d'outils. Tous les sélecteurs utiles restent accessibles directement dans le bandeau.

🆕 Fonctionnalités disponibles
  • Recherche par portrait (Photofit) : trouvez des patients ayant un portrait similaire grâce à la comparaison d'images
  • Scores de similarité : chaque fiche affiche un badge coloré indiquant le pourcentage de ressemblance
  • Recherche par similarité : cliquez sur n'importe quel élément d'un patient pour trouver des cas similaires
  • Sélecteur de langue simplifié : clic direct sur le drapeau pour changer de langue
  • Sélecteurs toujours visibles : base de données, langue, mode de détection dans le bandeau

🚀 2. Premier démarrage

2.1 Écran d'accueil

2.4 Contrôles du bandeau

Sélecteur de base de données

Le menu déroulant [Base ▼] permet de choisir la base de données à interroger. Le sous-titre sous "Search" indique la base active et le nombre de patients.

Sélection de la langue 🌐

Cliquez sur le bouton langue pour ouvrir le panneau de sélection. La langue s'applique immédiatement au clic sur le drapeau.

  • Langues disponibles : FR 🇫🇷, EN 🇬🇧, DE 🇩🇪, ES 🇪🇸, IT 🇮🇹, PT 🇵🇹, PL 🇵🇱, RO 🇷🇴, TH 🇹🇭, AR 🇸🇦, CN 🇨🇳, et AUTO (détection automatique)
  • Checkbox "→fr" : force la traduction des résultats en français, quelle que soit la langue de la question

Mode de détection

Le menu déroulant [Détection ▼] permet de choisir le moteur d'analyse des requêtes (standard, IA...). Le mode "standard" analyse les mots-clés localement, tandis que les modes IA envoient la question à un modèle de langage pour une interprétation plus fine.

💡 Bon à savoir Le thème jour/nuit est sauvegardé automatiquement dans le navigateur. Il peut aussi suivre les préférences système (mode "auto").

🔍 3. Utilisation de base

3.1 Effectuer une recherche

  1. Cliquez dans la zone de recherche au centre de l'écran
  2. Saisissez votre requête en langage naturel
  3. Lancez la recherche en cliquant sur ⬆️ ou en appuyant sur Entrée
💡 Exemples de requêtes patients avec béance • classe II division 1 • adultes avec encombrement • enfants de moins de 12 ans

3.2 Recherche vocale 🎤

Le bouton microphone permet deux modes :

Action Mode Description
Clic simple Normal Écoute unique, recherche après 5 sec de silence
Clic long Mains libres Écoute prolongée, dites "Kitview" + votre recherche
Mode mains libres Dites "Kitview patients avec bruxisme de moins de 30 ans" pour lancer automatiquement la recherche.

3.3 Panneau latéral

Le panneau gauche propose :

  • ✨ Nouvelle recherche : Réinitialise l'interface
  • Conversations récentes : Historique des recherches (cliquez pour copier, ▶ pour relancer)
  • Exemples : Suggestions de recherches prédéfinies
💡 Astuce La bordure droite du panneau est redimensionnable par glisser-déposer.

3.4 Affichage des résultats

Les résultats s'affichent sous forme de cartes patient contenant :

  • ID patient (en haut à gauche de la carte)
  • Portrait (photo ou silhouette)
  • Prénom (fond clair) et Nom (fond sombre)
  • Sexe (♂/♀, fond clair) et Âge (fond sombre)
  • Pathologies groupées par tag (voir section 3.5)

Cliquez sur une carte pour voir les détails complets du patient.

🎯 3.5 Recherche par similarité - "Même X que..."

🆕 Fonctionnalité V5.1 — Enrichie en V5.2 (Photofit) Cette fonctionnalité permet de trouver des patients similaires à un patient de référence en cliquant directement sur ses caractéristiques. En V5.2, la recherche par portrait compare visuellement les photos des patients.

Principe général

Après une recherche initiale, tous les éléments des cartes patients sont cliquables. Cliquez sur n'importe quelle caractéristique pour trouver d'autres patients ayant la même.

Comment utiliser cette fonctionnalité

Étape 1 : Recherche initiale

Effectuez une recherche normale, par exemple : patients avec bruxisme

Étape 2 : Sélection du patient de référence

Cliquez sur n'importe quel élément cliquable d'une fiche patient :

Élément cliqué Recherche générée
Portrait 📸"même portrait" — comparaison Photofit avec scores de similarité (V5.2)
Prénom"même prénom"
Nom"même nom"
Sexe (♂/♀)"même sexe"
Âge"même âge" (±3 ans)
Tag (ex: Béance)"même béance"
Pathologie complète"même béance antérieure gauche modérée"

Étape 3 : Patient sélectionné

La fiche du patient de référence s'affiche avec un fond jaune pour la distinguer :

ID: 1234   Portrait | Guillaume Moulin   ♂ 32 ans

Béance Béance Antérieure Gauche Modérée

Étape 4 : Ajouter des critères

Cliquez sur d'autres éléments du même patient pour combiner les critères.

Exemple : après avoir cliqué sur "portrait", cliquez aussi sur "âge" → la recherche devient "même portrait et même âge que Guillaume Moulin"

Étape 5 : Retirer un critère

Les critères actifs sont entourés d'une bordure rouge. Re-cliquez sur un élément actif pour le retirer :

Béance ← Critère actif (bordure rouge)    Classe II ← Critère inactif

Étape 6 : Changer de patient de référence

Cliquez sur n'importe quel élément d'un autre patient : le premier patient sera désélectionné et le nouveau deviendra la référence.

Étape 7 : Revenir à la recherche initiale

Retirez tous les critères actifs en re-cliquant dessus, ou lancez une nouvelle recherche.

Syntaxe directe dans la barre de recherche

Vous pouvez aussi taper directement des requêtes "même" :

même portrait que Guillaume Moulin même âge et même sexe que Hélène Joly mêmes portrait, prénom et âge que Jean Dupont même béance antérieure que id 123

Affichage des pathologies

Les pathologies utilisent un système de groupement visuel :

Type Apparence Exemple
Tags (1 mot) Fond sombre, texte blanc Béance
Pathologies (complètes) Fond clair, texte noir Béance Antérieure Gauche Modérée

Les pathologies sont groupées par tag : toutes les pathologies commençant par "Béance" sont affichées ensemble sous le tag "Béance".

💡 Astuce Si vous cliquez sur une pathologie complète alors que le tag correspondant était sélectionné, le tag sera automatiquement désélectionné (la pathologie est plus précise).

Recherche par portrait (Photofit) 📸

🆕 Nouveauté V5.2 La recherche par portrait utilise la comparaison d'images pour trouver des patients ayant un visage similaire.

Comment lancer une recherche par portrait

Deux façons de déclencher la recherche :

  1. Cliquer sur la photo d'un patient dans les résultats → lance automatiquement même portrait que [Prénom Nom]
  2. Taper dans la barre de recherche : même portrait que Guillaume Moulin ou même portrait que id 123

Scores de similarité

Chaque fiche patient affiche un badge coloré devant le mot PATHOLOGIES indiquant le pourcentage de ressemblance avec le portrait de référence :

Badge Score Signification
🎯 100% 100% Patient de référence (fond jaune)
🟢 97% ≥ 80% Ressemblance excellente
🔵 72% ≥ 60% Bonne ressemblance
🟠 54% < 60% Ressemblance modérée

Lecture des résultats

Les résultats d'une recherche par portrait sont triés par score décroissant. Le patient de référence apparaît en premier avec un fond jaune et le badge 🎯 100%.

Le bandeau bleu en haut des résultats indique la recherche en cours, par exemple :

même portrait que Thierry Lecoq base01600.db / standard
💡 Astuce Après une recherche par portrait, vous pouvez affiner en cliquant sur d'autres critères du patient de référence (âge, sexe, pathologies...) pour combiner les filtres.
⚠️ Important Les recherches "même" ne sont pas enregistrées dans l'historique des conversations récentes, car elles sont généralement des explorations temporaires.

Raccourci Action

4.2 Astuces

  • Autocomplétion : Le champ de recherche propose des suggestions basées sur l'historique
  • Panneau redimensionnable : Glissez la bordure droite du panneau latéral
  • Cartes patients : Cliquez pour développer, cliquez à nouveau pour réduire
  • Thème automatique : Le mode "Auto" suit les préférences système
  • Recherche "même" : Cliquez sur n'importe quel élément d'un patient pour trouver des cas similaires
  • Photofit : Cliquez sur la photo d'un patient pour trouver des portraits ressemblants (V5.2)
  • Langue instantanée : Cliquez directement sur un drapeau pour changer de langue

🔍 5. Recherche dans le guide

🆕 Nouveauté Une barre de recherche full-text permet de trouver rapidement n'importe quel mot ou expression dans ce mode d'emploi.

5.1 Utilisation

La barre de recherche 🔍 « Chercher dans le guide... » est située dans le bandeau supérieur.

  1. Tapez un mot ou une expression (minimum 2 caractères)
  2. Les occurrences sont surlignées en jaune dans la page
  3. L'occurrence courante est surlignée en orange
  4. Utilisez les boutons ◀ ▶ pour naviguer entre les résultats
  5. Le compteur indique votre position (ex: 3/12)

5.2 Raccourcis

RaccourciAction
F3Résultat suivant
Shift+F3Résultat précédent
EscapeEffacer la recherche

🤖 6. Assistant IA

🆕 Nouveauté Un assistant IA est disponible pour répondre à vos questions sur le mode d'emploi. Il utilise le contenu de cette page comme contexte pour fournir des réponses précises.

6.1 Accéder à l'assistant

Cliquez sur le bouton bleu 💬 en bas à droite de l'écran pour ouvrir la fenêtre de chat.

6.2 Poser une question

  1. Tapez votre question dans le champ de saisie en bas de la fenêtre de chat
  2. Appuyez sur Entrée ou cliquez sur le bouton d'envoi
  3. L'assistant analyse le contenu du mode d'emploi et vous répond

6.3 Questions suggérées

Des puces de suggestions sont proposées pour vous guider. Cliquez dessus pour lancer la question automatiquement.

6.4 Fonctionnement technique

L'assistant utilise un modèle d'IA (Claude ou GPT) hébergé côté serveur via des variables d'environnement sécurisées. Aucune clé API n'est stockée côté navigateur.

💡 Bon à savoir

L'assistant fonctionne uniquement lorsque le serveur est accessible et qu'une clé API (ANTHROPIC_API_KEY ou OPENAI_API_KEY) est configurée côté serveur. Si l'assistant n'est pas disponible, un message d'erreur s'affiche.
