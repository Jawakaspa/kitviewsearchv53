Comportement

Paramètres de fonctionnement et limites
⏱️

Durée du cycle de démonstration

Définissez l'intervalle en secondes entre chaque recherche automatique en mode démo. Une valeur plus élevée laisse plus de temps pour voir les résultats.
Intervalle entre les recherches
secondes
🎤

Mode mains libres continu

Le mode mains libres reste à l'écoute en permanence. Dites "Kitview" suivi de votre recherche pour lancer une requête sans toucher le clavier - idéal pendant les manipulations.
Délai d'écoute après détection
secondes
📊

Nombre maximum de résultats

Limitez le nombre de patients affichés pour améliorer les performances. Une limite plus basse accélère l'affichage des résultats.
Limite de patients affichés
patients
📄

Patients par page (pagination)

Définissez combien de patients sont affichés par page avant de devoir paginer. Une valeur plus petite permet un chargement plus rapide de chaque page.
Nombre par page
patients
⚠️

Gestion des erreurs

Configurez l'affichage des erreurs techniques. Par défaut, les erreurs sont masquées et l'application affiche simplement "0 patient trouvé". Activez ces options pour voir les messages d'erreur en cas de problème.
Options d'affichage des erreurs
📋 Comportement selon les options :
• Aucune case → "0 patient trouvé" (silencieux)
• Afficher erreurs → "Erreur technique" + détails en console F12
• Les deux cases → Message d'erreur complet affiché dans la page

Configuration des options de détection automatique
Sélectionnez les types de données que l'IA doit détecter automatiquement dans vos questions. La commande CLI correspondante est générée automatiquement.
